# Environments

## In a nutshell

Use the API base URL provided by the Subscriptions Engine team for your workspace. These docs use placeholder URLs so examples are easy to adapt.

## Base URLs

| Environment | Base URL |
| --- | --- |
| Local/demo | `http://localhost:8080` |
| Production placeholder | `https://api.example.com` |

Replace `https://api.example.com` with the real base URL for your deployment.

## Authentication per environment

API keys are environment-specific when separate deployments are provided. Keep sandbox/demo keys separate from production keys.

```bash
curl https://api.example.com/v1/customer/ \
  -H "X-Api-Key: sub_xxx"
```

## Sandbox testing

A full hosted sandbox key model is not currently exposed in the public API. For demos, use the API base URL and Nomba test credentials provided by the platform team.

See [Sandbox Testing](/operations/sandbox-testing) for a recommended test flow.
